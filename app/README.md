# Parkalyzer

A Python CLI application that calculates average walking distance and duration from residential census grid cells to parks and greenspaces in German cities, using OpenStreetMap data and the openrouteservice routing engine.

Built as the demonstration application for the [OSM Workshop 2026](../).

---

## How it works

Parkalyzer combines three data sources:

1. **OpenStreetMap** — park and greenspace polygons (`leisure`, `landuse`, `poi` layers via osmprj)
2. **German census grid data** — 100 m × 100 m population grid cells (`Zensus 2022`, imported via zensus2pgsql)
3. **openrouteservice** — foot-walking routes between each census cell and nearby parks

The workflow is:

```
parks find       →   routes calculate   →   report
(import parks)       (compute routes)       (show results)
```

Results include per-city and per-district breakdowns of the share of residents within 5, 15, and 30 minutes walk of a park.

---

## Requirements

- [pixi](https://pixi.sh) — all other dependencies are managed through it
- A running PostgreSQL instance with PostGIS (started by `scripts/dev-setup.sh`)
- OSM data loaded via [osmprj](https://github.com/travishathaway/osmprj)
- German census data loaded via [zensus2pgsql](https://github.com/travishathaway/zensus2pgsql)
- A running [openrouteservice](https://openrouteservice.org) instance (started by [ors-launcher](https://github.com/travishathaway/ors-launcher))

## Setup

### 1. Clone and enter the app directory

```bash
git clone git@github.com:travishathaway/osm-workshop-2026.git
cd osm-workshop-2026/app/
```

### 2. Start a pixi shell

```bash
# Start a pixi shell with all dependencies
pixi shell -e dev
```

### 3. Start all services

The `dev-setup.sh` script starts PostgreSQL, downloads and imports Brandenburg OSM data, imports German census data, and starts openrouteservice:

```bash
bash scripts/dev-setup.sh

# Load environment variables (DSN, ORS URL, schema names)
source env.sh
```

This takes several minutes on first run (OSM import + ORS graph build). Subsequent runs are fast — already-loaded data is detected and skipped.

## Usage

All commands accept `--dsn` and `--ors-url` flags (or the `PARKALYZER_DSN` / `PARKALYZER_ORS_URL` environment variables set by `env.sh`).

### Import parks from OSM

```bash
parkalyzer parks find Berlin
```

Use `--dry-run` to preview the count without writing to the database:

```bash
parkalyzer parks find Berlin --dry-run
```

List imported parks:

```bash
parkalyzer parks list
```

### Calculate walking routes

Compute walking routes from every census grid cell to all parks within 3 km:

```bash
parkalyzer routes calculate Berlin
```

Check progress:

```bash
parkalyzer routes status
```

Use `--limit N` to process only a subset of pairs (useful for testing):

```bash
parkalyzer routes calculate Berlin --limit 500
```

### Generate a report

```bash
parkalyzer report Berlin
```

## Configuration

All settings can be supplied as environment variables or CLI flags:

| Environment variable | CLI flag | Default | Description |
|---|---|---|---|
| `PARKALYZER_DSN` | `--dsn` | *(required)* | PostgreSQL DSN (`postgresql+psycopg://...`) |
| `PARKALYZER_ORS_URL` | `--ors-url` | `http://localhost:8080` | openrouteservice base URL |
| `PARKALYZER_OSM_SCHEMA` | `--osm-schema` | `public` | PostgreSQL schema holding OSM data |
| `PARKALYZER_ZENSUS_SCHEMA` | — | `zensus` | PostgreSQL schema holding census data |
| `PARKALYZER_SRID` | — | `3857` | Spatial reference ID for geometry storage |

`source env.sh` (generated by `dev-setup.sh`) sets all of these for you.

## Project structure

```
app/
├── src/parkalyzer/
│   ├── commands/
│   │   ├── db.py        # parkalyzer db  — Alembic migration commands
│   │   ├── parks.py     # parkalyzer parks — find and list parks from OSM
│   │   ├── routes.py    # parkalyzer routes — async ORS route calculation
│   │   └── report.py    # parkalyzer report — accessibility report
│   ├── config.py        # Configuration dataclass (env vars + CLI overrides)
│   ├── constants.py     # Schema names, routing constants, buffer distances
│   ├── ors.py           # openrouteservice HTTP client
│   └── main.py          # CLI entry point
├── alembic/             # Database migration scripts
├── scripts/
│   ├── dev-setup.sh     # Start all services, import data, run migrations
│   └── dev-teardown.sh  # Stop all services
├── tests/
│   ├── unit/            # Fast unit tests (no database or network required)
│   └── integration/     # Integration tests (require live services)
└── pyproject.toml       # Project metadata, pixi environments, task definitions
```
